How to Configure back-end server in Silk Framework!




You have some methods to configure the back-end server, in the config file or diretly in the code.

Using a Config file

Can find the config file in 'Silk_Installation_Path'/Data/config.json

Configure Steep by Steep

Server
IP: Define a target IP of the server (not is a real Endpoint, just you require configure you Endpoint server to listen in this IP)
Port: Just the Port of the server
Database
Host: IP/File or Alloc of the database host
Port: Just the Port of the database
Name: Name definition, require for context implementations in Silk
Connection: The connection string for create the connection with the database
Loggin
EnableMonitor: Enable Silk monitor, the recomendation is disabled in production
EnableConsole: Enable log in console
Certificate
KeyPath: Key file of Certificate for HTTPS connections
CertPath: Certificate file for HTTPS connections
UniqueThread: Indicate if you want use unique thread for manage the server
Locale: Indicate to silk server, your Localization Key

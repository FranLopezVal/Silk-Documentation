How to Routing in Silk Framework!




The routing of Silk is very simple and easy to use.

Routes common info

Silk have three types of routes:

  • First one is Attribute Route.

    • You can implement the route via attribute in the target method what you want to expose.

    Methods Allowed
    GET
    POST
    [...Other allow in next version]

    In the above code, the expose route is "/products/json/:setprice" and the method is GET.

    The setprice is a parameter that you can pass to the method.

    When user request this method, it return a html object with the value of the parameter. You can return multiple types of objects like JSon, XML, HTML, FILE, etc.

  • Second one is Static Route.

    • You can implement the route via static files in the 'Silk_Installation_Path'/Assets folder. Silk will automatically detect the file and expose it to the user, you can find the monitor files in the ./Assets/silk/monitor folder. Can use like simple web server.

  • Third one is Dynamic Route.

    • The last one is the dynamic route, you can implement the route via the RouterManager class.

About Silk Framework!




Why develop it?

History: As a Full-stack developer focused on Front-end and a lover of design, I have always had the need to create fast Back-ends, what I find are giant back-ends to simply make a functional mock for my projects. If I use ASP the library is more complex than necessary, if I use Node.Js the project folder occupies 80 Mb (thanks node_modules) to return two JSON. WHY!?

I have always been a fan of C #, I have always loved the language, the syntax, the performance, and the community. So I decided to create a simple and lightweight framework for building web services in C #. I want to make it easy to use, while still providing the power and flexibility needed for complex applications.
